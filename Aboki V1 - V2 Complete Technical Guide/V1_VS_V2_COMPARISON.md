# Aboki V1 → V2 Architecture

## Part 4: What Truly Changes — V1 vs V2

| Category              | V1                     | V2                                         |
| --------------------- | ---------------------- | ------------------------------------------ |
| **Account Model**     | EOA                    | Smart Contract Account                     |
| **Authentication**    | Wallet signature       | Biometric + Passkeys                       |
| **Gas Fees**          | User pays              | Sponsor pays (gasless)                     |
| **Identity**          | 0x address             | @username / phone                          |
| **Balance Storage**   | External wallet        | Smart account                              |
| **Transaction Speed** | ~5 mins                | Instant (Aboki) / 90s (Bank)               |
| **User Relationship** | One-time transactional | Persistent daily use                       |
| **Network Effects**   | None                   | Strong                                     |
| **Revenue Model**     | FX spread only         | Hybrid fees, merchant fees, float, premium |
| **Market**            | Crypto-savvy           | Anyone with a smartphone                   |
| **Retention**         | Low                    | High                                       |
| **Viral Coefficient** | Near zero              | High                                       |

---

#  Summary

V1 is a **bridge**.  
V2 is a **financial ecosystem**.
